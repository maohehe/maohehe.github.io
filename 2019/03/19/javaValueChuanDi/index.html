java值传递与引用传递？

发表于 2019-03-19 | 分类于 java ， java基础 | 评论数： | 阅读次数：
本文字数： 2.5k | 阅读时长 ≈ 2 分钟

首先明确一下，java中没有引用传递，只有值传递！

基本类型的传递

基本数据类型作为参数传递给方法时，传递的都是变量的副本。

示例，

与引用类型不同，基本数据类型的值都是存放在变量里的，而引用类型栈中存放变量，堆中存放对象实例，栈中变量的值是堆中对象的地址值。

当一个基本数据类型的变量作为参数传递给一个方法时，是拷贝一个变量值，传递给方法中的一个变量，实际操作的是方法中的这个变量，而原来的那个变量是根本没有改变的。

引用类型的传递

除了八大基本数据类型之外，其他的都是引用类型了

  • 类
  • 接口
  • 数组
  • 集合

上面说了，引用类型在栈和堆中都有占用内存，对于Person p = new Person();来说，首先new Person()在堆中开辟一块内存用来保存这条指令创建的对象，然后在栈中保存一个变量p，这个变量中保存的就是堆中创建的那个对象的地址，也就是常说的，p指向堆中的那块地址。

如果一个方法将引用类型作为参数的话，传递的p变量的一个副本，也就是将p变量指向的地址复制一份传递给方法的参数指定的那个变量。在方法中其实是对该变量进行操作，由于是传递的地址值，所以相应的改变在原来的变量那里也会显示出来。
