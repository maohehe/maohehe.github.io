主线程

main程序是程序入口，对编译好的class文件执行时，就可以运行这个编译好的程序。

这个过程中，系统自动创建了一个进程（程序）和一个线程（main线程）。

也就是说，系统运行就会自动创建主线程main。

创建线程

Thread类

任务定义好了就需要创建线程来执行任务了。

Java中Thread类用来创建线程，每个Thread对象都代表一个线程。

Thread类有几种常见的构造方法，来实例化线程对象，

由上面的构造方法可以发现，创建线程只有通过实例化出Thread对象来创建。而Thread类实现了Runnable接口，所以也可以覆写run()方法来在Thread类中定义任务。

所以根据任务的定义方式，创建线程又可分为两种：

  1. 通过实现Runnable接口定义任务，再将任务传给Thread来创建对象。
  2. 直接继承Thread类，实例化出对象，在类中的run()方法中直接定义任务。

Thread类的start()方法用来启动线程的执行。

对于创建出来的线程对象，必须调用其start()方法之后，才有可能执行。

线程相关方法

Thread提供不少方法方法我们获取线程信息、控制线程等。

  1. 获取、设置线程名称

    • String getName()
    • void setName(String name)

  2. 获取线程ID

    • long getId()，系统会自动为每个线程分配一个long类型的ID。

  3. 获取、设置线程优先级(Priority

    • int getPriority()
    • void setPriority()

    Java优先级由1~10表示，十个等级，Thread中有三个常量来表示常用的等级，

    然而我们最好不要使用这种方式去修改线程优先级，线程执行的前后顺序还是交给处理器去自行调度为好。

  4. 线程睡眠(sleep())

    • static void sleep(long millis) ，指定毫秒数的睡眠
    • static void sleep(long millis,int nanos)，指定毫秒数和纳秒数的睡眠

    哪个线程调用该方法，那个线程就会睡眠。

    该方法会抛出InterruptException异常，需要自行捕获（catch）

  5. 获取当前正在执行线程(currentThread())

    • static Thread currentThread()，返回一个线程对象，是当前线程

    这是一个静态方法，在哪个线程中调用Thread.currentThread()方法，返回的就是这个线程对象。

  6. 守护线程(Daemon)

    • boolean isDaemon()，查询线程是否是守护线程
    • void setDaemon(boolean on)，设置线程为守护线程

    守护线程就是后台线程，专门为普通线程提供服务，例如jvm专门负责垃圾收集的线程就是守护线程，如果所有普通线程都死掉了，那守护线程也没必要活下去了。

    注意：只有未启动的线程(未调用start方法)才可以设置为守护线程(setDaemon)。还有从普通线程中创建的线程是普通线程，从守护线程中创建的线程是守护线程。

  7. 让出本次时间片(yield())

    • static void yield()，让出本次时间片

    线程的执行都是靠处理器调度，每次分配一定的时间片给线程执行，时间片结束就重新调度分配。

    yield()方法就表示线程放弃了此次时间片，处理器立即进行重新调度。

    当然处理器可能恰巧又调度到了这个线程，这也说不准。

  8. 等待另一线程结束(join())

    • void join()，等待线程终止才继续执行
    • void koin(long millis)，指定的时间内等待线程终止，时间结束，无论是否终止，都继续执行
    • void join(long millis,int nanos)，同上

    有时候需要在一个线程中等待另一个线程执行完成之后，本线程才能继续执行。
