JDBC基础

JDBC概念

概述

JDBC（Java DataBase Connectivity）是一种用于执行SQL语句的Java API，可以为多种关系数据库提供统一访问，它由一组用Java语言编写的类和接口组成。

JDBC是sun公司定义的一套接口规范。JDBC对Java程序员而言是API，对实现与数据库连接的服务提供商而言是接口模型（规范）。作为API，JDBC为程序开发提供标准的接口，并为数据库厂商及第三方中间件厂商实现与数据库的连接提供了标准方法。

也就是说，对于多种关系型数据库（Mysql，DB2，Oracle等），通过使用JDBC这一种方式就可以访问他们，而不用对每种数据库都编写一套程序。而实际的访问代码是由数据库厂商根据JDBC接口规范来实现的。

数据库驱动

各大数据库厂商编写的JDBC接口的实现类，他们将实现类编译后打包成jar包并发布，这就是数据库驱动，即我们使用JDBC连接Mysql数据库，当我们调用JDBC中接口方法时，实际调用Mysql驱动的实现类。

获取数据库操作对象

数据库操作对象用来执行SQL语句，并返回执行的结果。

有两种常用的statement对象：

  1. Statement：由createStatement()创建，用于发送简单的SQL语句（不带参数）。
  2. PreparedStatement：继承自Statement接口，由preparedStatement创建，用于发送含有一个或多个参数的SQL语句。PreparedStatement对象比Statement对象的效率更高，并且可以防止SQL注入，所以我们一般都使用PreparedStatement。
    • 然而通过preparedStatement()方法创建PreparedStatement对象时就要立即传入SQL语句。
